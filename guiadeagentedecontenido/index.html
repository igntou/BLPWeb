1

Instalá Claude Desktop

Claude Desktop es la app de escritorio de Anthropic. Es gratis y permite conectar herramientas externas como Canva y Airtable directamente desde tu computadora.

  1. Andá a claude.ai/download desde tu navegador y descargá la versión para tu sistema (Mac o Windows)
  2. Abrí el instalador y seguí los pasos — igual que cualquier otra app
  3. Iniciá sesión con tu cuenta de Anthropic. Si no tenés, creá una gratis en el mismo paso
  4. Dentro de Claude Desktop, hacé clic en el ícono "+" en la barra inferior y seleccioná "Conectores"
  5. Buscá Canva, hacé clic en Conectar y autorizá con tu cuenta de Canva
  6. Repetí el proceso para Airtable — autorizá con tu cuenta de Airtable

Plan gratuito: Los conectores de Canva y Airtable están disponibles sin costo. El plan Pro ($20/mes) solo es necesario para el Bonus de automatización con Cowork al final de esta guía.

2

Creá el Project y pegá el system prompt

Un Project en Claude es un espacio con memoria e instrucciones permanentes. El system prompt es el cerebro de tu agente — le dice quién es, qué puede hacer y cómo comportarse.

Cómo crear el Project

  1. En Claude Desktop, hacé clic en Projects en la barra lateral izquierda
  2. Hacé clic en Nuevo Project y ponele el nombre: "Agente de Contenido"
  3. Dentro del Project, hacé clic en el ícono de configuración y seleccioná Project Instructions
  4. Pegá el system prompt completo que está abajo y guardá
  5. Listo — cada conversación dentro de este Project va a usar estas instrucciones automáticamente

System prompt completo — pegá esto en Project Instructions

System prompt
## ROL
Sos mi Agente de Contenido personal. Tu trabajo es
ayudarme a crear contenido auténtico y personalizado
para hacer crecer mi negocio en redes sociales —
sin que suene genérico ni de IA.

## CONTEXTO DEL NEGOCIO
[Se carga automáticamente desde el PDF
"Mapa de Contenido™ + Brand Reference" en los
archivos del Project. Si no encontrás ese archivo,
ejecutá el PROTOCOLO DE ONBOARDING antes de hacer
cualquier otra cosa.]

## PROTOCOLO DE ONBOARDING
### Solo si no existe el PDF en los archivos

FASE 1 — Mapa de Contenido™
Hacé estas preguntas UNA POR UNA:
P1: ¿Cómo se llama tu negocio y qué hacés exactamente?
P2: ¿A quién le vendés? Edad, género, dónde vive,
    qué hace, qué le preocupa.
P3: ¿Cuál es el problema más grande que le resolvés?
    Usá sus palabras, no las tuyas.
P4: ¿Qué resultado concreto logra después de trabajar
    con vos? Sé específico: tiempo, dinero, cambio.
P5: ¿Cuáles son tus servicios o productos principales?
    Nombre, qué incluye, precio si querés.
P6: ¿Por qué vos y no otro? ¿Qué te hace diferente?
P7: ¿En qué redes publicás y con qué frecuencia ideal?
P8: ¿Tu meta con el contenido es más seguidores,
    más ventas, o las dos cosas?

FASE 2 — Brand Reference
Luego de las 8 preguntas, pedí esto:
"Ahora necesito tu identidad visual para que el
contenido sea 100% tuyo:
1. Tu paleta de colores (hex si tenés)
2. Tus fuentes o estilo tipográfico
3. Una imagen de ejemplo de cómo querés que
   se vean tus posts — subila al chat o describila
4. ¿Qué NUNCA debe aparecer en tu contenido?"

FASE 3 — PDF unificado
Con toda la info, generá el documento con este formato
y ofrecé descargarlo como PDF:

───────────────────────────
MAPA DE CONTENIDO™ + BRAND REFERENCE
Negocio: [nombre] | Fecha: [fecha]
───────────────────────────
SOBRE EL NEGOCIO
→ Qué hace: [P1]
→ Cliente ideal: [P2]
→ Problema que resuelve: [P3]
→ Resultado que genera: [P4]
→ Servicios/productos: [P5]
→ Diferenciador: [P6]
→ Redes activas: [P7]
→ Meta de contenido: [P8]

MIS 5 PILARES DE CONTENIDO
Pilar 1 — [nombre]: [qué resuelve] | [formatos]
Pilar 2 — [nombre]: [qué resuelve] | [formatos]
Pilar 3 — [nombre]: [qué resuelve] | [formatos]
Pilar 4 — [nombre]: [qué resuelve] | [formatos]
Pilar 5 — [nombre]: [qué resuelve] | [formatos]

BRAND REFERENCE
→ Paleta: [colores hex]
→ Tipografía: [fuentes o estilo]
→ Tono de voz: [cómo suena la marca]
→ Estilo visual: [descripción del look]
→ Canva Brand Kit ID: [ID si existe / Pendiente]
→ Nunca usar: [restricciones visuales o de tono]
→ Referencia visual: [descripción del post ejemplo]

PROMPT BASE CANVA
"Diseñá un [tipo] con: colores [paleta],
fuente [tipografía], estilo [descripción],
tono [tono de voz]. Referencia: [post ejemplo].
Formato: instagram_post 1080x1350px."
───────────────────────────

Al terminar indicá:
"Descargá este PDF y subilo a los archivos del
Project (ícono de clip en la barra superior).
Desde ese momento lo leo automáticamente en
cada sesión."

## HERRAMIENTAS Y PERMISOS

CANVA:
→ Crear carruseles, posts e imágenes
→ Siempre usar el Brand Kit ID del PDF si existe
→ Si no existe, usar paleta y estilo del Brand Reference
→ Mostrar textos de cada slide antes de crear el diseño
→ Crear diseño solo después de aprobación del texto
→ Si el usuario tiene Canva Pro, guiarlo para crear
  el Brand Kit directamente en Canva

AIRTABLE:
→ Crear registros nuevos de contenido
→ Actualizar campos: Status, Caption, Fecha, Link Canva
→ Eliminar registros con confirmación explícita del usuario
→ Leer el calendario para contexto al generar nuevas ideas
→ Tabla esperada: Nombre | Pilar | Tipo | Fecha |
  Caption | Status | Link Canva | Notas

## REGLAS DE OPERACIÓN

SIEMPRE:
→ Leer el PDF antes de generar cualquier contenido
→ Mostrar el output antes de guardar o publicar
→ Esperar "aprobado" antes de escribir en Airtable
→ Incluir exactamente 5 hashtags por caption
→ Cada post: título, pilar, tipo, fecha, caption + CTA
→ No repetir el mismo pilar dos días seguidos

NUNCA:
→ Inventar datos del negocio si no están en el PDF
→ Guardar en Airtable sin aprobación explícita
→ Crear contenido genérico que no refleje el
  Brand Reference
→ Usar más de 5 hashtags en Instagram

SI FALTA CONTEXTO:
→ Sin PDF en el Project: ejecutar onboarding
→ Dato faltante específico: preguntar, no inventar

## FORMATO DE RESPUESTA
→ Directo y orientado a la acción
→ Contenido presentado listo para usar
→ Al final de cada output preguntar:
  "¿Aprobamos y guardamos en Airtable?"
3

Sesión 1 — Onboarding del agente

Solo la primera vez · 15-25 minutos

El agente detecta que no hay PDF y ejecuta el onboarding automáticamente. Al final de esta sesión tu agente ya sabe todo sobre tu negocio y tiene tu estilo visual listo para usar.

Primer mensaje 8 preguntas Brand visual PDF unificado Subir al Project

Primer mensaje para activar el onboarding

Copiá esto y envialo
Hola, es nuestra primera sesión.
No hay PDF en el Project todavía.
Ejecutá el protocolo de onboarding completo.

Qué pasa en cada fase

  1. Fase 1 — Mapa de Contenido™: El agente te hace 8 preguntas una por una sobre tu negocio, cliente ideal, problema que resolvés, servicios y diferenciador. Respondé con tus propias palabras.
  2. Fase 2 — Brand Reference: El agente te pide tu paleta, fuentes y una imagen de ejemplo de cómo querés que se vean tus posts. Podés subir una imagen al chat o describirla con palabras.
  3. Fase 3 — PDF unificado: El agente genera el documento completo con tu Mapa de Contenido™, tus 5 pilares, tu Brand Reference y el prompt base de Canva ya configurado con tus colores y estilo.
  4. Subir al Project: Descargás el PDF y lo subís a los archivos del Project (ícono de clip en la barra superior). El agente lo lee automáticamente en todas las sesiones siguientes.

Para la imagen de referencia: Subí cualquier imagen al chat — un post tuyo que te guste, una referencia de otra cuenta, o una captura de pantalla. El agente la analiza y la convierte en descripción de estilo visual para Canva.

Al terminar la sesión 1 tenés en tu Project

  • PDF "Mapa de Contenido™ + Brand Reference" — la memoria permanente del agente
  • Prompt base de Canva ya configurado con tus colores, fuente y estilo dentro del PDF
  • Agente listo para crear contenido personalizado desde la próxima sesión
4

Prompts para usar cada semana

Sesión 2 en adelante

El agente lee el PDF automáticamente. Ya sabe quién sos, a quién le hablás y cómo se ve tu marca. Estos prompts son los que usás semana a semana.

Generar calendario de 2 semanas

Calendario semanal
Generá el calendario de contenido para las próximas
2 semanas, empezando el [fecha de inicio].
- 1 post por día, lunes a viernes
- Variá entre los 5 pilares sin repetir el mismo
  dos días seguidos
- Mezclá: educación, entretenimiento y prueba social
- Para cada post: título, pilar, tipo de contenido,
  fecha, caption con CTA y exactamente 5 hashtags

Mostrámelo primero antes de guardar en Airtable.

Generar un post específico

Post puntual
Necesito un [Carrusel / Reel / Post imagen] del
pilar "[nombre del pilar]" para el [fecha].

El ángulo que quiero: [describí la idea o escribí
"proponé vos basándote en mi avatar"].

Caption completo con hook, cuerpo, CTA y 5 hashtags.

Aprobar y guardar en Airtable

Confirmación
Aprobado. Guardá este post en Airtable,
tabla "Contenido", con Status = "Aprobado".
Confirmame cuando esté guardado.
5

Crear en Canva y gestionar Airtable

El agente crea los diseños con tu estilo visual y gestiona todos los cambios en Airtable — incluyendo editar, actualizar estado o eliminar registros.

Crear carrusel en Canva

Prompt carrusel
Quiero crear el carrusel "[nombre del post]" en Canva.

Primero mostrámelos textos de cada slide:
- Slide 1: Hook — frase corta que pare el scroll
- Slides 2 al 5: Un punto del caption por slide
- Slide 6: CTA — [tu llamada a la acción]

Cuando apruebe los textos, creá el diseño en Canva
usando el Brand Reference y el prompt base del PDF.

Crear post de imagen simple

Prompt post imagen
Creá en Canva el post imagen para "[nombre del post]".
Usá el Brand Reference del PDF para los colores y estilo.
Mostrámelo antes de guardar el link en Airtable.

Gestionar registros en Airtable

Actualizar / eliminar
En Airtable, en el registro "[nombre del post]":

Opciones — decí cuál querés:
→ Cambiá el Status a [Aprobado / Publicado / Idea]
→ Actualizá el caption: [nuevo texto]
→ Cambiá la fecha a: [nueva fecha]
→ Guardá el link del diseño de Canva: [link]
→ Eliminá este registro

Confirmame cuando esté hecho.

Canva Pro: Si tenés Canva Pro, el agente puede crear tu Brand Kit directamente en Canva con tus colores y fuentes. Pedíselo con: "Creá mi Brand Kit en Canva con los datos del Brand Reference del PDF."

6

Publicar y cerrar el ciclo

La publicación es manual — vos descargás y publicás. El agente actualiza el estado en Airtable para que tu calendario siempre esté al día.

  • Descargá el diseño desde Canva en JPG o PNG
  • Copiá el caption desde Airtable — ya viene con los 5 hashtags
  • Publicá en tu red social en la fecha indicada
  • Pedile al agente que cambie el Status a "Publicado" en Airtable

Marcar como publicado

Prompt de cierre
Publiqué el post "[nombre del post]".
Actualizá en Airtable: Status → "Publicado".
Confirmame cuando esté hecho.

Actualizar el Brand Reference si cambiás tu estilo

Prompt actualización de marca
Quiero actualizar mi Brand Reference.
[Describí el cambio: nuevo color, fuente,
imagen de referencia o estilo]

Generá una versión actualizada del PDF con estos
cambios para que lo vuelva a subir al Project.

Bonus — Automatizá con Claude Cowork

Requiere Claude Pro · $20/mes

Claude Cowork ejecuta tareas automáticas en tu computadora aunque no tengas Claude abierto. Con esto tu agente genera ideas nuevas cada lunes y te manda el resumen del calendario cada viernes — solo.

Mini-Bonus A — Ideas semanales automáticas a Airtable

Tarea semanal — lunes 9am
Configurá esta tarea en Claude Cowork para que se
ejecute cada lunes a las 9:00am:

"Abrí el Project 'Agente de Contenido' y ejecutá:

Generá 5 ideas nuevas de contenido para esta semana,
una por pilar. Para cada idea incluí: título, pilar,
tipo de contenido y fecha sugerida. Variá los formatos
(no repitas el mismo tipo dos veces seguidas).

Guardalas en Airtable con Status = 'Idea'.
Cuando estén guardadas, enviame un resumen a
[tu email] con los títulos y pilares de las 5 ideas."

Mini-Bonus B — Resumen semanal del calendario por correo

Tarea semanal — viernes 6pm
Configurá esta tarea en Claude Cowork para que se
ejecute cada viernes a las 6:00pm:

"Abrí el Project 'Agente de Contenido' y ejecutá:

Revisá mi Airtable, tabla 'Contenido' y hacé un resumen:
1. Posts publicados esta semana (Status = Publicado)
2. Posts aprobados para la próxima semana (Status = Aprobado)
3. Ideas pendientes de revisar (Status = Idea)

Enviame el resumen a [tu email] con el asunto:
'Resumen de contenido — semana del [fecha]'"

Cómo activar Cowork: Claude Desktop → menú superior → Cowork → Nueva tarea → pegá el prompt → configurá frecuencia (semanal), día y hora → Guardar. Cowork ejecuta la tarea automáticamente.
